Game Design Document

Full design specification: mechanics, player flows, scoring, DDA, and therapeutic goals.

1Overview

Garden Grow is a 2-player motion-based gardening game using webcam hand tracking. Players restore a garden together, with invisible AI ensuring both a grandmother with dementia and her teenage grandson stay engaged and challenged.

Key Differentiators

  • Camera-based input — no controllers, no wearables, no special hardware
  • Real-time Dynamic Difficulty Adjustment (DDA) per player independently
  • Co-op and competitive modes on a single shared screen
  • Fully synthesized audio — zero file dependencies
  • Runs in any modern browser — hosted on GitHub Pages for instant play

Core Values

  • No failure states — every action produces positive feedback
  • Gentle, intuitive controls that feel natural and forgiving
  • Social connection through shared play experiences
  • Therapeutic benefit embedded in every interaction
  • Adaptive difficulty that meets each player where they are

2Design Pillars

🏋️
Physical Activity
Hand tracking exercises map to therapeutic movements — reaching, grasping, pouring, and hovering. Every gardening action doubles as a gentle physical exercise.
🧩
Cognitive Stimulation
Multi-step tasks (plant, nurture, harvest) engage pattern recognition, sequential actions, and short-term memory in a low-pressure context.
🤝
Social Engagement
2-player modes encourage conversation, shared attention, and helping behaviors. Playing together strengthens bonds between generations.
🌿
Wellbeing Support
Calming garden theme, positive reinforcement, celebratory endings, and zero failure states create a safe, joyful experience.

3Target Audience

👵
Primary: Person Living with Dementia
Mild to moderate cognitive impairment. Benefits from repetitive positive actions. The game must be self-evident — no instructions needed to start playing.
👦
Secondary: Companion
Teenager, family member, or caregiver playing alongside. Provides encouragement and shares the experience. The game keeps them engaged too.
🩺
Tertiary: Clinical Staff
Uses the game in care settings. Needs minimal setup (<30 seconds). Values observable engagement metrics and session data for care plans.

5Game Modes

🧑
Solo Mode
Full-width garden with 2–3 plant pots. Standard challenge progression. One player, one hand, full therapeutic benefit.
🌱
Community Garden (Co-op)
Shared garden with a shared goal. The Magic Pumpkin bonus triggers when both players touch it simultaneously — confetti burst and timer pause.
🏆
Village Fair (Competitive)
Split zones with individual scoring. Rubber-band mechanic keeps it fair. Blue and Red ribbons awarded — never a “game over.”

6Fair Play DDA System

The AI monitors each player independently every 5 seconds. It adjusts game parameters silently so that both players — regardless of ability — feel engaged and successful.

Adjustment Parameters

ParameterRangeEffect
seedSpeed0.4x – 1.8xHow quickly new seeds appear
hitBoxMultiplier0.8x – 2.0xSize of interactive touch zones

Struggling Player Support

  • Fewer than 2 harvests per 10 seconds: needs deplete 40% slower
  • Idle for more than 3 seconds: hit boxes grow up to 70% larger
  • Auto-hint arrows appear pointing toward the next action

Rubber-Band Mechanic

When the score gap exceeds 20% in competitive mode, the trailing player receives a Golden Watering Can that instantly maxes all plant needs — a gentle catch-up boost that feels like a reward, not a consolation.

8Accessibility

👁️
Visual
Minimum 32px text, WCAG AA contrast ratios, color-independent indicators, no rapid flashing, and 70px+ hit zones for all interactive elements.
✋
Motor
Both hands tracked with generous timing windows. Seated play fully supported. No fine motor precision required — broad gestures work.
🧠
Cognitive
Single visible objective at a time. Consistent, predictable UI patterns. No penalties for mistakes. Auto-help system guides without judgment.

2Why Data Matters

Objective measurement replaces subjective observation. Traditional therapeutic assessments rely on clinician notes and patient self-reporting, both of which introduce bias and inconsistency. Garden Grow provides quantified metrics captured automatically during every session — reaction times measured in milliseconds, range of motion mapped to screen coordinates, and engagement durations tracked to the second.

Longitudinal tracking reveals patterns invisible in individual sessions. A single therapy session tells you how a patient performed today. Weeks and months of data reveal trends: is reaction time improving? Is session duration gradually extending? Are idle periods becoming less frequent? These trajectories are the real clinical signal, and they require consistent, automated data collection to detect.

Evidence-based care plan adjustments. When the DDA curve shows a patient consistently performing above their difficulty ceiling, that is clinical evidence for increasing therapeutic challenge. When engagement metrics decline over several sessions, that data supports a care plan modification. Garden Grow transforms gameplay into clinical evidence that supports treatment decisions.

4Privacy & Security

Patient data protection is built into the architecture from the ground up.

  • 🔒
    All data stored locally. Session data persists in the browser's localStorage on the facility's own device. No data leaves the machine without explicit action.
  • 🚫
    No cloud transmission. The current version operates entirely offline. No analytics services, no third-party trackers, no external API calls.
  • 🔐
    Future: encrypted sync (opt-in). Planned cloud sync will use end-to-end encryption and require per-facility opt-in consent. Facilities retain full data ownership.
  • 🇪🇺
    GDPR-ready architecture. Data structures are designed for right-to-erasure compliance. Patient records can be fully deleted at any time with no residual data.

5Dashboard Integration

All collected data feeds directly into the Clinician Dashboard — a real-time analytics interface designed for care providers. The dashboard presents session trends, motor engagement heatmaps, DDA difficulty curves, and exportable progress reports.

Clinicians can filter by patient, compare sessions over time, and generate summary reports for care plan reviews — all without leaving the browser.
